Dataset Creation and Processing Pipeline

Automated Data Mining and Preprocessing Pipeline

This page describes the data mining and preprocessing pipeline used to build the fish species dataset.

The dataset creation process was time-consuming and well-suited for automation. An automated pipeline was developed to mine image data, process images, and organize the dataset. This approach reduces manual effort while maintaining dataset quality. The pipeline is modular and easily customizable, allowing new data sources to be incorporated and image processing methods to be applied across the entire dataset.

Data Sources

  • iNaturalist — community-sourced wildlife observations with labeled images
  • GBIF (Global Biodiversity Information Facility) — aggregated biodiversity occurrence records with image data

Pipeline Overview

  1. Download images by species using automated scripts (iNaturalist + GBIF)
  2. Organize images into species-specific folders
  3. Manually filter images (keep / reject / maybe)
  4. Isolate high-quality images into a clean dataset
  5. Apply background removal and normalization
  6. Resize and standardize images for model training
  7. Assemble final dataset for machine learning pipeline

Manual Filtering Criteria

After automated image collection, each image was manually reviewed before entering the clean dataset. This step was necessary because online image sources often include poor angles, partial fish, obstructed subjects, duplicate images, low-resolution images, or images containing multiple species.

  • Keep: clear image of the target species, full or mostly full fish visible, good enough resolution, and minimal obstruction.
  • Reject: wrong species, poor visibility, severe blur, heavily obstructed fish, duplicate image, or image not useful for training.
  • Maybe: uncertain species label or usable image quality but not ideal for the final dataset.

Only images marked as acceptable were moved forward into the clean dataset and used for later preprocessing or machine learning.

Background Removal Comparison

Two dataset preparation paths were evaluated: one using raw images and one using background removal. Background removal produces a controlled, high-contrast dataset, while raw images preserve environmental context.

For this project, the raw image dataset was used for field-oriented machine learning, since it better represents real-world underwater conditions. The background-removed dataset was retained as a controlled alternative for potential future model training and comparison.
