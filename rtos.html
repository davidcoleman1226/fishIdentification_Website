RTOS Underwater Image Recovery

Real-Time Feedback Control System for Underwater Image Enhancement

This page presents the real-time underwater image recovery system implemented for the CS 766 fish identification project. The system uses a spatial RGB gain map with temporal feedback smoothing to reduce color distortion, turbidity effects, and low-visibility artifacts before machine learning inference.

Mathematical Foundation

Each video frame is represented as an RGB vector field:

\[ I_k(i,j) = \begin{bmatrix} R_k(i,j) \\ G_k(i,j) \\ B_k(i,j) \end{bmatrix} \]

The local mean RGB value over each spatial tile is:

\[ \bar{I}_{k,m,n} = \frac{1}{|\Omega_{m,n}|} \sum_{(i,j)\in \Omega_{m,n}} I_k(i,j) \]

The raw spatial gain is computed as:

\[ G^{\text{raw}}_{k,m,n} = \frac{I_{\text{target}}} {\bar{I}_{k,m,n} + \epsilon} \]

The raw gain is clipped to avoid excessive amplification or attenuation:

\[ G^{\text{clipped}}_{k,m,n} = \text{clip} \left( G^{\text{raw}}_{k,m,n}, G_{\min}, G_{\max} \right) \]

Temporal feedback smoothing is applied using the previous gain map:

\[ G_{k,m,n} = \alpha G_{k-1,m,n} + (1-\alpha)G^{\text{clipped}}_{k,m,n} \]

The smoothed gain map is expanded back to full image resolution:

\[ G_k(i,j) = \text{resize} \left( \text{blur}(G_{k,m,n}) \right) \]

The corrected image is produced by applying the gain map element-wise:

\[ I^{\text{corrected}}_k(i,j) = I_k(i,j) \odot G_k(i,j) \]

Variable Definitions

  • \(k\): current video frame index or time step.
  • \(i,j\): pixel row and column indices in the full-resolution image.
  • \(m,n\): spatial tile row and column indices in the downsampled image.
  • \(I_k(i,j)\): RGB vector at pixel location \((i,j)\) in frame \(k\).
  • \(R_k(i,j)\): red channel intensity at pixel \((i,j)\).
  • \(G_k(i,j)\): green channel intensity at pixel \((i,j)\) when used inside the image vector.
  • \(B_k(i,j)\): blue channel intensity at pixel \((i,j)\).
  • \(\Omega_{m,n}\): set of pixels inside spatial tile \((m,n)\).
  • \(|\Omega_{m,n}|\): number of pixels inside tile \((m,n)\).
  • \(\bar{I}_{k,m,n}\): mean RGB vector measured inside tile \((m,n)\) during frame \(k\).
  • \(I_{\text{target}}\): desired target RGB vector used as the color-balance reference.
  • \(\epsilon\): small positive constant used to prevent division by zero.
  • \(G^{\text{raw}}_{k,m,n}\): raw spatial RGB gain vector for tile \((m,n)\).
  • \(G_{\min}\): minimum allowable gain value.
  • \(G_{\max}\): maximum allowable gain value.
  • \(G^{\text{clipped}}_{k,m,n}\): raw gain after limiting it between \(G_{\min}\) and \(G_{\max}\).
  • \(G_{k,m,n}\): temporally smoothed gain value for tile \((m,n)\).
  • \(G_{k-1,m,n}\): previous frame's smoothed gain value for tile \((m,n)\).
  • \(\alpha\): temporal feedback smoothing coefficient.
  • \(1-\alpha\): weighting applied to the newly computed gain.
  • \(G_k(i,j)\): full-resolution gain map value applied at pixel \((i,j)\).
  • \(\odot\): element-wise multiplication between RGB vectors.
  • \(I^{\text{corrected}}_k(i,j)\): corrected RGB output pixel after gain compensation.

Pseudocode

Input: video frame I_k
Output: corrected frame I_corrected

1. Convert frame from BGR to RGB.
2. Normalize RGB values to [0, 1].
3. Downsample the frame for faster computation.
4. Divide the downsampled frame into spatial tiles.
5. Compute the mean RGB vector in each tile.
6. Compute the raw spatial gain using the target RGB vector.
7. Clip gain values between G_min and G_max.
8. Apply temporal feedback smoothing using the previous gain map.
9. Blur and upsample the gain map to full frame resolution.
10. Multiply the original frame by the full-resolution gain map.
11. Apply percentile normalization.
12. Convert the corrected frame back to BGR.
